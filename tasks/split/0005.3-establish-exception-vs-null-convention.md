# Subtask 0005.3: Establish Exception vs Null Convention

**Parent Task**: 0005 - Exception Hierarchy  
**Effort**: S (1h)  
**Dependencies**: 0005.1, 0005.2

## Description

Establish and document a clear convention for when to throw exceptions vs return null.

## Actions

- [ ] Define convention: exceptions for invalid input/invariants, null for optional values
- [ ] Review current code against convention
- [ ] Document convention in `docs/exceptions.md` (create file)
- [ ] List any violations that need fixing
- [ ] Document guidelines for contributors

## Acceptance Criteria

- Convention clearly defined
- Convention documented in docs/exceptions.md
- Current code reviewed against convention
- Violations listed with recommendations
- Contributor guidelines included

## Definition of Done

- [ ] Convention established and documented
- [ ] Current code reviewed against convention
- [ ] Violations listed
- [ ] docs/exceptions.md started
- [ ] Changes committed to version control

## Tests Needed

None directly, but convention will guide test creation in other subtasks.
