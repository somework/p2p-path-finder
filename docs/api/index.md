## Search state signatures

Search frontier de-duplication relies on compact signature strings that describe the search
context. Signatures are composed of one or more segments separated by the pipe (`|`) delimiter.
Each segment must contain a non-empty label and value separated by a colon (`:`). Labels may not
contain `|` or `:` characters, and values may not contain `|`. The formatter trims leading and
trailing whitespace from each segment so runtime logs remain normalised.

Common segments include:

- `range` – always present; encodes the spending interval as
  `CURRENCY:MIN_AMOUNT:MAX_AMOUNT:SCALE` or `null` when no range is known.
- `desired` – always present; encodes the target spend amount as `CURRENCY:AMOUNT:SCALE` or
  `null` when unbounded.
- `signature` – optional; records the discovered route signature when heuristics capture it.
- `pruned` – optional; indicates whether pruners disabled the state (`true` or `false`).
- Additional labels may appear when bespoke guard rails attach diagnostic metadata.

For logging and diagnostics, `SomeWork\P2PPathFinder\Application\PathFinder\Search\SearchStateSignatureFormatter`
provides a `format()` helper that converts either a `SearchStateSignature` instance or its string
value into an ordered associative map of `{label => value}` pairs.
