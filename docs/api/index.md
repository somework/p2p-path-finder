## Numeric precision helpers

All monetary/tolerance value objects now store their amounts as
`Brick\Math\BigDecimal` instances and expose `decimal()` accessors alongside the
existing numeric-string getters. When integrating with lower layers
(`SearchState`, `CandidatePath`, `PathCost`, etc.) prefer the decimal accessors to
avoid re-normalising strings; serialization helpers such as
`PathResultFormatter` continue to emit canonical numeric strings so response
payloads remain backwards compatible.
